Watch your real-time data

Splunk Particle is an experiment in visualizing Splunk real-time search results. The particle animation enables you to immediately see patterns in your data as they occur.


Examples

Twitter client activity - see the distribution of clients connecting to Twitter as seen from the sample stream. The most common clients have icons associated with them.

Tagged tweets - see the tweets that have hashtags, their client type, and the raw tweet.

Splunk indexing lag - see how long it takes for data to show up in Splunk, the host it came from, and the breakdown between data that is and is not eventtyped.

Tcpdump traffic - see real-time network traffic from the packet level, split by network layer, protocol, and hardware vendor.

Splunk source pianoroll - see how Splunk receives data in both streamed and buffered-write mode.


Quick Start

To try out the particle visualizations:

  1. Install Splunk 4.2 (this is not compatible with 4.1 or older)
  2. Install the SplunkViz app from Splunkbase
  3. Restart Splunk
  4. Select the "Splunk Visualizations" app from the home screen

At this point, you should be able to browse to the SplunkViz app home page, and view the example Splunk-specific visualizations shown above.

To use the tcpdump example: (warning: this generates a very high volume of data)

  1. Install Splunk on a linux box
  2. Have tcpdump installed on your server
  3. Verify that the scripted input correctly references your tcpdump:
    $SPLUNK_HOME/etc/apps/SplunkViz/bin/capture_tcpdump.sh
  4. Restart splunk as root (tcpdump requires it to sniff network traffic)
  5. Go to Manager > Data Inputs > Scripts
  6. Edit the entry for capture_tcpdump.sh eth1 and replace the eth1 with the network interface of your machine
  7. Save the entry
  8. Click the "enable" link to start capturing data
  9. Return to the search interface and navigate to Saved Searches > Tcpdump - general visualization
  10. Be sure to disable the input when you're done

To use the Twitter examples:

  1. Install Splunk on a linux box
  2. Obtain a single-use access token from Twitter at dev.twitter.com. You'll need all four of the following parameters that are unique to your account. The last two can be generated once you register an application.
    • consumer key
    • consumer secret
    • access token
    • access token secret
  3. Copy the file:
    $SPLUNK_HOME/etc/apps/SplunkViz/default/tweepy.conf
    to:
    $SPLUNK_HOME/etc/apps/SplunkViz/local/tweepy.conf
  4. Edit the copied file and fill in the four Twitter tokens
  5. Restart Splunk
  6. Go to Manager > Data Inputs > Scripts
  7. Click the "enable" link next to the capture_twitter.sh item to start capturing Twitter data

Components

Particles

Particles are the visual representation of discrete events that exist in a data set. These particles are generally small icons that can retain their resolution in a high density display. Currently, particles can be rendered as dots, field value labels, or images.

Emitters

Emitters are the source of particles and are responsible for producing particles that are displayed in the system. Emitters are free to produce any number of particles, in any form. Emitters must be paired with at least one collector in order to establish a visualization. There is always a root emitter that generates all of the particles.

Collectors

Collectors are the destination for particles and are responsible for identifying particles they want to collect (based on user-provided criteria). For any emitter, there are one or more collectors assigned in a priority list. For each particle emitted, each collector is given the opportunity to receive the particle.

Renderer

The renderer is the master controller that is responsible for animating particles generated by emitters and linking them with their respective collectors. For each emitter, there are one or more collectors that can be attached. Once a particle is emitted from an emitter, each of the collectors are asked, in order, if it wants to collect the particle. If so, then the renderer will animate and render the particle as defined by the user. Uncollected particles will be dimmed and flow off-screen.

Field Splitters

Field splitters are the main point of user interaction in the Particle visualization. They are composed of a single collector and emitter, and render the field value of the particles that enter its collector through text, image, frequency histogram, or some combination of those three. Splitters also define how emitted particles are rendered.

Motivations

Humans are not adept at evaluating large masses of raw data. Particle visualization is a way to represent large masses of data in such a way that is more comprehensible through the following means:

  • Homogenize and iconify all events. In order to visualize large data sets in a single screen yet retain resolution on the discrete events, each event must be reduced to an abstract representation using small shapes or icons. Homogenizing those representations allows the user to quickly group like items and also assess the pattern of the entire data set.
  • Animate events based on arrival-order. Since time is a primary dimension in real-time search, rendering events such that their spatial relationship in 2D or 3D space is proportional to their arrival-order allows users to easily comprehend the time distribution of events.
  • Illustrate significant events. Humans excel at detecting visual pattern anomalies so altering event representation allows users to efficiently identify significant events. Users should be able to add criteria that flags events as significant.

Known Issues

Browser performance

Performance of the Splunk Particle Flash movie varies wildly among different platforms. We are still trying to isolate the performance issue, and will update this page when more information is available. Listed below are our observations on CPU load.

OS/browser combinations that consume a low percentage of CPU (~25%)

  • OSX + FF3
  • Win7 + IE9

OS/browser combinations that consume a high percentage of CPU (~75%)

  • OSX + Chrome
  • OSX + FF4
  • Win7 + Chrome
  • Win7 + FF3
  • Win7 + FF4
